# Security Policy

## Supported Versions

We actively provide security updates for the following versions of `marn-express-initializer`:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of our tools and the projects they generate very seriously. If you discover a security vulnerability within this CLI, please do not report it via public GitHub issues.

Instead, please follow these steps:

1. **Email us**: Send a detailed report to **security@codebiruni.com** (or your personal professional email).
2. **Describe the issue**: Include steps to reproduce the vulnerability, the potential impact, and any suggested fixes.
3. **Response time**: You can expect an acknowledgment of your report within 48 hours.

## Our Process

Once a vulnerability is reported:

1. We will verify the issue and assess its severity.
2. We will work on a fix in a private environment.
3. A new patched version (e.g., a "patch" release like `1.0.3`) will be published to NPM.
4. We will credit you for the discovery (if you wish) in the release notes.

Thank you for helping keep the **Robius Sani** ecosystem safe!
